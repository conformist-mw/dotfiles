# Cloudflare migration — service worksheet

Цель: разложить каждый сервис по корзине, чтобы из этого вывести CF DNS/tunnel, фаервол,
политики Access и правки в Ansible-ролях. Заполни/поправь поля после `→`. Я проставил догадки.

## Легенда корзин

- **WAF**      — публично за Cloudflare WAF, авторизация своя (логин самого приложения). Семья = аккаунт в app.
- **ACCESS**   — публично, но за Cloudflare Access (auth на эдже, замена oauth2-proxy). Указать «кому».
- **PUBLIC**   — полностью открыто, auth не нужна (для кого угодно).
- **VPN**      — доступно только из WireGuard, наружу не публикуется.
- **INTERNAL** — бэкенд/демон, не смотрит на пользователя вообще.
- **ARCHIVE**  — выпиливаем, доказано что не нужен.

Поля для заполнения:
- `keep`   → KEEP или ARCHIVE
- `bucket` → одна из корзин выше (поправь мою догадку)
- `who`    → кому нужен доступ (только я / семья / конкретные люди — имена-роли / все)
- `bypass` → пути, которые должны быть открыты мимо Access (вебхуки, публичные эндпоинты)
- `note`   → любые пояснения
